# 打包构建优化（dist 体积）

组件库按需引入，避免全量打包

图片压缩 + 小图（<10KB）转为 base64

开启 Gzip/Brotli 压缩，体积减少 60%~70%（需 Nginx/CDN 配合）

利用 Tree-Shaking 移除未使用代码

构建时移除 console、注释、debugger，清理未使用 CSS

代码分割，避免单个 JS 体积过大，提升加载与解析速度

使用打包分析工具（webpack-bundle-analyzer /visualizer）定位体积大户



# 运行时渲染优化（DOM & 卡顿）
虚拟列表 / 虚拟表格
使用防抖、节流，减少频繁触发的事件与接口请求
减少重排重绘，避免频繁操作 DOM 样式


# 资源加载优化
懒加载 预加载：路由/组件
图片原生懒加载 loading="lazy"


# 网络请求优化
合并接口请求，减少 HTTP 次数
接口分页，避免一次性返回大量数据



# 缓存优化
前端缓存：LocalStorage / SessionStorage / IndexedDB
接口缓存：Redis 缓存 
静态资源缓存：强缓存、协商缓存
