
# 高并发防数据库击穿：FastAPI 完整落地方案（思路+代码）

## 一、核心思路（标准工业级）

1. **入口限流拦截**：超出数据库/服务处理能力的请求直接拒绝，从源头防击穿
2. **读请求**：**Redis 缓存 + 缓存预热**，大幅减少数据库查询
3. **写请求**：**消息队列削峰**，异步平滑写入数据库
4. **兜底**：数据库扩容（主从/读写分离）

---

## 二、关键概念解释

1. **超出请求能力**：自定义阈值（如每秒5请求），超过即为过载
2. **拦截**：限流组件直接返回「系统繁忙」，不进业务/数据库
3. **缓存+预热**：热点数据放Redis，服务启动时提前加载缓存
4. **消息队列**：写请求先入队，异步消费入库，防数据库写爆
5. **数据库扩容**：主从架构+读写分离，分担库压力

---

## 三、FastAPI 代码实现（可直接运行）

### 1. 限流拦截（防过载核心）

```bash
pip install slowapi limits
```

```python
from fastapi import FastAPI, Request
from slowapi import Limiter, _rate_limit_exceeded_handler
from slowapi.util import get_remote_address
from slowapi.errors import RateLimitExceeded

limiter = Limiter(key_func=get_remote_address)
app = FastAPI()
app.state.limiter = limiter
app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)

# 每秒最多5请求，超出直接拦截返回429
@app.get("/api/query")
@limiter.limit("5/second")
async def query_data(request: Request):
    return {"data": "查询成功"}
```

### 2. 读请求：Redis 缓存 + 预热

```bash
pip install redis
```

```python
import redis
app = FastAPI()
redis_client = redis.Redis(host="localhost", port=6379, db=0, decode_responses=True)

# 缓存查询：先查Redis，无数据再查库并回写缓存
@app.get("/api/goods/{id}")
def get_goods(id: str):
    cache_key = f"goods:{id}"
    cache_data = redis_client.get(cache_key)
    if cache_data:
        return {"source": "缓存", "data": cache_data}
    # 模拟库查询
    db_data = f"商品{id}信息"
    redis_client.setex(cache_key, 600, db_data) # 缓存10分钟
    return {"source": "数据库", "data": db_data}

# 缓存预热：启动时加载热点数据
@app.on_event("startup")
async def startup_event():
    hot_ids = ["1001","1002","1003"]
    for id in hot_ids:
        redis_client.setex(f"goods:{id}", 600, f"商品{id}信息")
```

### 3. 写请求：消息队列削峰（RabbitMQ）

```bash
pip install aio-pika
```

```python
# 写入接口：只入队，不直接写库
import aio_pika
@app.post("/api/order")
async def create_order(user_id: str, money: float):
    msg = f"用户{user_id}下单{money}元"
    connection = await aio_pika.connect_robust("amqp://guest:guest@localhost/")
    channel = await connection.channel()
    await channel.default_exchange.publish(
        aio_pika.Message(body=msg.encode()), routing_key="order_queue"
    )
    await connection.close()
    return {"status": "排队成功"}

# 消费者脚本：异步平滑写库
import asyncio
async def consumer():
    connection = await aio_pika.connect_robust("amqp://guest:guest@localhost/")
    channel = await connection.channel()
    queue = await channel.declare_queue("order_queue", durable=True)
    async with queue.iterator() as q:
        async for message in q:
            async with message.process():
                print("写入数据库:", message.body.decode())
asyncio.run(consumer())
```

### 4. 数据库扩容：读写分离

```python
# 读走从库，写走主库
def read_db(sql): pass # 从库查询
def write_db(sql): pass # 主库写入

@app.get("/api/user")
def get_user(): return read_db("select * from user")
@app.post("/api/user")
def add_user(): return write_db("insert into user ...")
```

## 四、完整架构总结

```
用户请求 → 限流拦截（超量直接拒绝）
  ↓
查询 → Redis缓存+预热 → 极少查库
写入 → 消息队列削峰 → 异步平滑入库
  ↓
数据库：主从+读写分离（扩容兜底）
```
