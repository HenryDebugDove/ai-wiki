**Trae Skill（技能）** 是你给 AI 助手预设的一套**固定指令、角色、工作流和输出规范**，本质是把你的重复要求“固化”成 AI 的**肌肉记忆**，让它每次都精准按你的规则办事。


**Skill = 可复用、可命名、可触发的 Prompt 模板。**

### 一、Skill 的核心作用

- **省去重复指令**：不用每次都跟 AI 说“按XX规范”“检查XX问题”
- **输出高度一致**：团队/个人风格统一，AI 不再随意发挥
- **复杂流程自动化**：把多步骤任务（审查→重构→测试）封装成一键调用
- **精准触发**：AI 自动识别场景，或你手动 `@` 调用

### 二、3 种创建方式（实操）

#### 1. 对话一键生成（新手首选）

直接在聊天框发自然语言：

> **帮我创建一个【代码审查大师】Skill，审查JS/TS，重点查Eslint规范、安全漏洞、性能瓶颈、注释覆盖率，输出问题清单+修复代码+评分**

Trae 自动生成 `.trae/skills/代码审查大师/SKILL.md`，无需手动写文件。

#### 2. 手动创建（精细控制）

1. 打开：**设置 → 规则和技能 → 技能 → 创建**
2. 填：名称、描述、触发词、**详细指令**（角色+流程+规范+格式）
3. 保存（自动生成目录与 `SKILL.md`）

#### 3. 导入共享 Skill

下载别人的 Skill 文件夹 → **创建并导入文件夹**

### 三、Skill 使用场景 + 实战例子（直接复制可用）

#### 1. 开发规范类（最常用）

**Skill 名称**：React+TS 规范开发员
**触发词**：react ts 组件 开发
**SKILL.md 核心指令**：

```markdown
# 角色
资深React+TypeScript前端专家，严格遵循Ant Design最佳实践。

# 规则
- 用函数组件 + Hooks
- 强类型定义interface/props
- 拆分通用逻辑为自定义Hook
- 加清晰JSDoc注释
- 错误边界+loading/空状态
- 组件名大驼峰，文件小驼峰
- 输出：代码+注释+使用示例
```

#### 2. 代码质量类

**Skill 名称**：Git 提交规范助手
**触发词**：git commit 规范
**指令**：

- 按 Conventional Commits：`type(scope): subject`
- type：feat/fix/docs/style/refactor/test/chore
- 英文短句，首字母小写，不超50字符
- 自动生成符合规范的commit信息

#### 3. 文档写作类

**Skill 名称**：API 文档自动生成器
**触发词**：生成接口文档
**指令**：

- 输出Markdown
- 结构：接口地址、请求方式、请求头、请求参数（必填/类型/说明）、响应示例、错误码、备注
- 附可直接运行的curl示例

#### 4. 业务/效率类

**Skill 名称**：周报自动总结
**触发词**：生成周报
**指令**：

- 结构：本周完成、问题与解决、下周计划、心得反思
- 语言简洁、数据化、突出成果
- 适配工作群汇报格式

#### 5. 结合 MCP（Skill + 工具 = 超级自动化）

**Skill 名称**：数据库查询助手（+MCP）
**触发词**：查询数据库 sql
**指令**：

1. 分析需求，生成安全SQL（防注入）
2. 调用MCP工具 `db_query` 执行
3. 返回格式化表格+关键结论
4. 给出性能优化建议

### 四、如何调用 Skill

1. **自动触发**：聊天提到关键词（如“代码审查”），AI 自动用对应 Skill
2. **手动调用**：输入 `@Builder 代码审查大师` 强制激活

### 五、Skill vs MCP（区别）

- **Skill**：**纯指令/规则/工作流**（Prompt 封装），无外部调用
- **MCP**：**外部工具/能力**（读文件、调API、连数据库）
- **最佳实践**：**Skill + MCP 组合**——Skill 定流程，MCP 做执行

### 六、与 Cursor 兼容性

- **Skill 格式不通用**：Trae 是 `SKILL.md`，Cursor 是 `cursorrules`
- **但内容可迁移**：把指令文本复制过去即可
