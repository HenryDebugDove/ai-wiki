# 全流程整合精简版（PRD+原型+UI+AI工作流）

## 一、PRD 是什么

**PRD=产品需求文档**，用来明确产品**做什么、规则是什么、如何验收**。

| 优点               | 缺点                 | 典型业务场景                  |
| ------------------ | -------------------- | ----------------------------- |
| 需求统一、可追溯   | 文档易臃肿、更新滞后 | 电商退款、后台审核、APP新功能 |
| 支撑测试验收       | 易过度文档化忽略用户 | 新系统从零到一、ToB交付       |
| 降低跨团队沟通成本 | 频繁迭代易与产品脱节 | 权限、金额、审批等复杂逻辑    |

## 二、PRD vs 原型 vs UI（Figma）核心区别

| 环节                | 核心内容                     | 给谁看           | 表现形式       | 解决问题                 |
| ------------------- | ---------------------------- | ---------------- | -------------- | ------------------------ |
| **PRD**       | 逻辑、规则、字段、异常、验收 | 后端、测试、PM   | 飞书/语雀/Word | 定义业务规则与边界       |
| **原型**      | 页面结构、交互跳转、流程     | 前端、设计、客户 | 墨刀/Axure     | 展示页面长什么样、怎么点 |
| **UI(Figma)** | 视觉、配色、字体、组件、细节 | 前端、设计       | 高保真设计稿   | 最终视觉样式与交付标准   |

一句话总结：
**PRD定逻辑 → 原型定流程 → UI定视觉**

## 三、PRD 常用工具

| 工具            | 特点                       |
| --------------- | -------------------------- |
| 飞书/钉钉/语雀  | 在线协作强，适合互联网团队 |
| Axure/墨刀/摹客 | 原型+PRD一体，交互友好     |
| Word/WPS        | 传统企业对外交付           |
| Notion          | 模块化、可关联任务         |

## 四、专门生成 PRD 的 AI 工具 vs 通用提示词

| 方案                                            | 优点                           | 缺点                       | 适用场景                    |
| ----------------------------------------------- | ------------------------------ | -------------------------- | --------------------------- |
| **通用AI+提示词**(豆包/DeepSeek)          | 零成本、灵活、适配任意业务     | 质量依赖提示词，需人工校对 | 个人/小功能/快速初稿        |
| **专业PRD AI工具**(墨刀AI/小摹AI/ChatPRD) | 自带模板、原型联动、结构化更强 | 部分付费、复杂业务仍需修改 | 团队协作、B端系统、正式交付 |

> 结论：**小需求用提示词即可，正式项目用专业AI工具更高效**。

## 五、AI 全链路工作流：PRD → 原型 → Figma UI

### 1. 一句话需求 → PRD（AI 5分钟生成）

提示词模板：

```
你是资深产品经理，生成可落地PRD：
产品：XX系统
用户：XX角色
核心功能：XX
输出：背景目标→功能清单→业务流程→字段规则→异常处理→验收标准
```

### 2. PRD → 可交互原型（AI 10分钟）

工具：**墨刀AI / Pixso AI**
操作：粘贴PRD → AI自动拆页面、做布局、加交互 → 人工微调

### 3. 原型 → Figma UI（AI 10分钟）

- 墨刀/Pixso 一键导出 Figma
- Figma 插件：Galileo AI / First Draft
- 输入风格要求，AI生成高保真视觉

### 4. 最终交付（AI自动补齐）

- 自动生成测试用例
- 自动生成前端代码片段
- 自动标注、切图

## 六、完整示例：用户登录模块

1. **PRD**
   - 手机号11位，密码8-20位
   - 输错3次锁定15分钟
   - 验证码5分钟有效
2. **原型**
   - 画登录页、输入框、按钮、跳转逻辑
3. **UI(Figma)**
   - 按钮颜色、输入框样式、错误提示样式
4. **AI流**
   需求→AI写PRD→墨刀AI生成原型→导出Figma→AI优化UI
