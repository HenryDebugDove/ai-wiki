
# 一、获取数据的主流方式（降序）

| 排名 | 获取方式                 | 主流程度   | 优点           | 适用场景   | 业务例子   |
| ---- | ------------------------ | ---------- | -------------- | ---------- | ---------- |
| 1    | 接口直接获取（模拟用户） | ⭐⭐⭐⭐⭐ | 最快最稳定     | API网站    | 抓商品数据 |
| 2    | 浏览器爬虫（DOM获取）    | ⭐⭐⭐⭐   | 接近真实用户   | 强反爬网站 | 抓页面内容 |
| 3    | OCR图像识别              | ⭐⭐       | 可识别图片数据 | 图片数据   | 抓验证码   |

---

## ① 接口直接获取（最推荐）

通过抓接口获取数据

优点：

* 速度快
* 数据完整
* 最稳定

业务例子：

* 电商商品数据
* 新闻数据
* 招聘数据

例如：

```js
fetch("/api/list")
```

---

## ② DOM 渲染获取

浏览器加载后：

从页面 DOM 获取

适用：

* JS动态网站
* 无接口网站

业务例子：

* 抓商品列表
* 抓评论

例如：

```js
document.querySelector(".title").innerText
```

---

## ③ OCR 图像识别

用于：

* 图片验证码
* 图片文字

业务例子：

* 验证码识别
* 图片数据抓取

常用工具：

* Tesseract
* 百度
* 阿里云

# 四、浏览器爬虫技术对比

| 技术       | 优点   | 缺点         | 适用场景   |
| ---------- | ------ | ------------ | ---------- |
| Playwright | 稳定强 | 学习成本高   | 强反爬网站 |
| Puppeteer  | 简单   | 只支持Chrome | 动态网站   |
| Selenium   | 多语言 | 慢           | 传统网站   |

# 十、面试总结（简短版）

如果面试官问：

**爬虫获取数据方式有哪些？**

你可以回答：

> 常见获取数据方式包括接口直接获取、DOM解析获取以及OCR图像识别，其中接口获取最常用，浏览器爬虫适合强反爬网站，OCR适合图片数据。

如果问：

**反爬虫有哪些？**

> 常见反爬虫包括IP封禁、请求频率限制、验证码、行为检测、代码混淆、Debugger反调试等。
