### 一、Vite 为什么比 Webpack 快？（核心原理）

**一句话总结**：Vite 利用**浏览器原生 ESM** 实现「开发不打包、按需编译」，并用 **esbuild** 预构建依赖，从架构上比 Webpack 全量打包快得多。

#### 1. Webpack 为什么慢（传统打包模式）

- **全量打包启动**：启动时必须递归分析**整个项目依赖图**，全部编译、打包成 bundle 后才能运行。
- **模块越多越慢**：中等项目启动 20–30 秒，大型项目可达分钟级。
- **HMR 粒度粗**：改一个文件常要重新打包相关 chunk，更新慢。

#### 2. Vite 快在何处（三大核心）

- **① 开发环境：原生 ESM + 不打包、按需加载**

  - 浏览器支持 `<script type="module">`，直接用 `import/export`。
  - Vite 只做**中转服务器**：浏览器要哪个模块，Vite 才编译哪个。
  - 启动几乎**秒开**（毫秒级），项目变大速度也稳定。
- **② 依赖预构建（esbuild）**

  - 用 Go 编写的 **esbuild** 编译速度比 Webpack/Babel 快 **10–100 倍**。
  - 首次启动把 `node_modules` 转成 ESM、合并小文件、缓存（`.vite/deps`）。
  - 避免浏览器发几百个小请求。
- **③ 精准 HMR**

  - 基于 ESM：改哪个文件，**只更新该模块及最小依赖链**。
  - 热更新几乎实时（<100ms），与项目大小无关。

#### 3. 核心对比（开发模式）

| 特性               | Webpack            | Vite                    |
| :----------------- | :----------------- | :---------------------- |
| **启动方式** | 先全量打包 bundle  | 直接启动，按需编译      |
| **模块加载** | 打包后运行         | 浏览器原生 ESM 按需请求 |
| **依赖处理** | 全量编译           | esbuild 预构建 + 缓存   |
| **HMR 速度** | 慢（重打包 chunk） | 极快（单模块更新）      |
| **大型项目** | 越大约慢           | 速度稳定                |

### 二、现在最主流是 Vite 吗？（2026 现状）

**是，Vite 已是新项目主流、接近事实标准**。

- **市场地位（2026）**

  - **Vite**：npm 周下载 ~1800 万，市场占有率约 **45%**，新项目首选。
  - **Webpack**：仍大量存在于**老项目/遗产工程**，新项目极少选用。
  - **Rspack / Turbopack**：快速增长（Rust 编写），但生态与份额仍不及 Vite。
- **框架默认**

  - Vue 3 / Nuxt 3：**默认 Vite**。
  - React 生态：Create React App 被弃，主流用 **Vite + React** / Next.js（内置 Turbopack）。
  - Svelte / Solid / Qwik 等：**全部默认 Vite**。
- **结论**

  - **新项目：优先 Vite**（极速、开箱即用、生态完善）。
  - **老项目：Webpack 仍在维护，但逐步迁移 Vite**。

### 三、Vite 与浏览器的关系（核心依赖）

Vite **完全基于现代浏览器的原生 ES Module（ESM）能力**设计。

#### 1. 开发时：浏览器是「主角」

- Vite Dev Server 只做两件事：
  1. 把源码（TS/JSX/Vue 等）**实时转成 ESM** 给浏览器。
  2. 处理 HMR、代理、资源等增强能力。
- **浏览器负责**：解析 `import`、依赖追踪、模块缓存、执行代码。
- 必须是**现代浏览器**（Chrome ≥89、Firefox ≥89、Safari ≥15、Edge ≥89）。

#### 2. 生产时：打包（Rollup/Rolldown）

- 开发「不打包」≠ 生产「不打包」。
- 生产构建用 **Rollup（Vite 6 改用 Rust 写的 Rolldown）** 打包优化、分包、压缩。
- 产物仍是给浏览器运行的静态资源（JS/CSS/图片）。

#### 3. 一句话关系

**Vite = 浏览器 ESM 的「增强器 + 编译器」**：开发让浏览器原生加载模块，生产打包成浏览器可高效运行的优化包。

---

### 四、一句话速记

- **Vite 快**：**ESM 按需加载 + esbuild 预构建 + 精准 HMR**。
- **主流度**：**2026 年新项目绝对主流**，Webpack 退守老项目。
- **浏览器关系**：**开发靠浏览器原生 ESM，生产打包给浏览器运行**。
