用 Trae/Cursor 这类 AI IDE 开发，要同时保证 **效率、质量、速度**，核心是：**把 AI 当“全栈编码员+测试员+审查员”，人当“架构师+产品经理+把关人”**。

---

## 一、效率：让 AI 替你做 80% 重复/机械工作

### 1. 项目级配置：一次定义，全程生效

- 在项目根目录放：
  - `.trae/config.json`：技术栈、TS 规则、ESLint、命名规范、目录结构
  - `.trae/rules/*.md`：团队编码规范、安全规则、禁止写法
  - `skills/`：把常用角色/指令做成可复用 Skill（如 @react-expert、@nestjs-clean）

效果：AI 每次生成都**自动符合规范**，不用反复说明。

### 2. 用 Builder/SOLO 模式做“全流程自动”

- 新项目：一句话需求 → Trae 自动拆任务、生成项目骨架、路由、CRUD、Dockerfile
- 小功能：`生成XX模块，带TS、校验、单元测试、日志、错误处理`
- 改需求：直接让 AI **批量重构/迁移**（如 React 类组件转 Hooks）

### 3. 精准 Prompt 模板（直接用）

```
功能：XX
技术栈：React19 + TS + Tailwind + ReactQuery
规范：
- 组件纯函数、自定义hooks拆分
- 错误边界、Suspense
- 单元测试用例覆盖主流程+边界
- 注释关键逻辑、TODO 标记待优化
- 接口返回统一 { code, data, msg }
交付：可直接运行的完整文件
```

→ 可用率从 30% → 85%+

### 4. 跨文件/全项目上下文

- 用 `#文件名`、`#目录`、`#整个项目` 让 AI 理解全局依赖
- 报错直接把日志拖进去：`按优先级排序，给出修复顺序和原因`

## 二、质量：AI 生成 ≠ 直接上线，3 道防线

### 1. 规则锁死（事前）

- `.trae/rules` 强制：
  - TS 严格模式、禁止 any
  - 密码必须加密、SQL 防注入、XSS 过滤
  - 禁用危险 API（eval、unsafe-eval）
- AI 生成时**自动校验**，不合规直接重写

### 2. 分层审查（事中）

1. **AI 自审**：让 AI 自己 Review
   > 检查这段代码：安全漏洞、性能、内存泄漏、边界case、可维护性
   >
2. **人工必审**（核心模块）
   - 业务逻辑、权限、支付、用户数据相关 **必须人写/人审**
   - AI 只做 CRUD、工具类、UI、配置、测试
3. **自动化检查**
   - ESLint、Prettier、TypeCheck 全量跑
   - 单元测试覆盖率 ≥80%（AI 生成 + 人工补）

### 3. 测试驱动（AI 帮你写）

- 先写接口/函数定义
- 让 AI：`生成XX的单元测试，覆盖正常/异常/边界/空值/超时`
- 先跑通测试，再写业务 → 质量稳很多

## 三、交付速度：流程化 + 人机分工

### 我的标准工作流（6A 简化版）

1. **Align（对齐）**：写清晰 Spec（功能、输入输出、验收标准）
2. **Architect（架构）**：人定技术方案、表结构、API 文档
3. **Atomize（拆任务）**：AI 拆小任务，逐个生成
4. **Approve（审批）**：人审架构/设计文档
5. **Automate（执行）**：AI 批量生成代码、测试、文档
6. **Assess（验收）**：跑测试、联调、修复 → 交付

### 人机分工黄金原则

- **你负责**：
  - 需求理解、架构决策、技术选型
  - 核心业务、复杂算法、安全、权限
  - Code Review、最终验收、上线把控
- **AI 负责**：
  - 样板代码、CRUD、配置、工具函数
  - 单元测试、接口文档、注释
  - Bug 定位、简单修复、性能建议

## 四、实战：我一天怎么干（示例）

- **9:00**：写 Spec + 架构设计 → 1h
- **10:00**：Trae Builder 生成项目骨架 + 基础模块 → 30min
- **10:30**：AI 生成 CRUD + 接口 + 测试 → 1h
- **11:30**：人工写核心业务逻辑 → 1h
- **14:00**：AI 查 Bug、优化、补测试 → 30min
- **14:30**：人工 Review + 修复 + 联调 → 1h
- **15:30**：AI 生成文档、部署脚本 → 30min
- **16:00**：测试、部署、交付

**结果**：

- 传统 3～5 天 → **1 天完成**
- 缺陷率下降 **30%～50%**
- 人只做 **20% 高价值工作**

## 五、三者关系一句话总结

- **效率** = 用好 Skill + 规则 + 精准 Prompt → AI 少返工
- **质量** = 规则锁死 + 分层审查 + 测试先行 → 上线少事故
- **速度** = 流程化 + 人机清晰分工 → 快速交付、可复制
