
# 1. K8s 是什么？

**K8s = Kubernetes**
就是一个**容器集群管理工具**，用来管一堆 Docker 容器。

你可以把它理解成：

- Docker 是**集装箱**
- K8s 是**集装箱码头调度中心**

它帮你干这些事：

- 把应用自动部署到多台服务器
- 应用挂了自动重启
- 流量大了自动加实例
- 流量小了自动减实例
- 某台服务器坏了，自动把应用迁走
- 统一管理日志、监控、网络、存储

一句话总结：
**K8s 让你把整个机房当成一台电脑用。**

# 2. 核心优缺点（表格）

| 优点                          | 缺点                         |
| ----------------------------- | ---------------------------- |
| 自动自愈：应用崩了自动重启    | 概念极多，学习曲线陡         |
| 自动扩缩容，应对流量洪峰      | 小项目（单服务器）完全没必要 |
| 不停机发布、回滚              | 配置复杂，yaml 容易写错      |
| 多机器统一管理，像一台机器    | 网络、存储、权限配置繁琐     |
| 微服务必备，适合大规模项目    | 本地跑 K8s 非常吃资源        |
| 多云/混合云通用，不绑定云厂商 | 排查问题难度比普通部署高     |

# 3. 真实业务场景（超多案例）

我给你举**前端/后端都能看懂的真实业务场景**。

## 场景 1：电商秒杀 / 618 / 双11

- 平时订单服务：2 个实例够用
- 秒杀开始，流量瞬间涨 20 倍
- K8s 自动扩容到 30 个实例
- 活动结束自动缩回到 2 个
  **不用人工干预，不崩、不浪费钱。**

## 场景 2：服务器突然宕机

- 某台物理机死机了
- 上面跑了 5 个微服务
- K8s 自动发现节点挂了
- 立刻在其他机器重新启动这 5 个服务
  **用户完全无感知。**

## 场景 3：前端项目自动化部署（真实前端场景）

你的 Vue/React 项目打包成 Docker 镜像后：

- 提交代码 → GitLab CI 构建镜像
- 自动推送到镜像仓库
- K8s 自动拉取最新镜像
- 滚动更新，**不停机发布**
- 发布出问题，一键回滚

## 场景 4：微服务太多，人工管不过来

一个系统可能有：

- 用户服务
- 商品服务
- 订单服务
- 支付服务
- 购物车服务
- 优惠券服务
  ...共 40+ 服务

人工一台台部署、重启、查看状态会疯。
**K8s 统一管理所有服务。**

## 场景 5：测试环境 / 预发 / 生产环境统一

以前：

- 开发环境：正常
- 测试环境：报错
- 生产：又不一样

用 K8s：
一套 yaml 配置，三环境完全一致，**从根源解决环境不一致问题**。

## 场景 6：公司上云（阿里云 / 腾讯云 / AWS）

不用改代码，只改少量配置，就能在任意云之间迁移。
**不被云厂商绑架。**

# 4. 最简单可运行代码示例（前端也能看懂）

## 示例 1：部署一个 Nginx（最简单应用）

```yaml
# nginx.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-nginx
spec:
  replicas: 3      # 启动 3 个实例
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:alpine
        ports:
        - containerPort: 80
```

运行：

```bash
kubectl apply -f nginx.yaml
```

效果：
K8s 会自动在集群里找空闲机器，启动 3 个 Nginx。

## 示例 2：把服务暴露出去，让外部访问

```yaml
# service.yaml
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
  - port: 80
    targetPort: 80
  type: NodePort
```

运行：

```bash
kubectl apply -f service.yaml
```

现在你可以通过 IP 访问 Nginx 了。

## 示例 3：查看运行状态

```bash
kubectl get pods
kubectl get deployment
kubectl get service
```

## 示例 4：自动扩容（应对高并发）

```bash
kubectl scale deployment my-nginx --replicas=10
```

瞬间变成 10 个实例抗流量。

# 5. 用生活类比再帮你彻底理解

- **Docker = 外卖餐盒**
  把饭菜打包，去哪都一样吃。
- **K8s = 外卖调度系统**

  - 订单多了自动加骑手
  - 骑手摔了自动换人送
  - 某区域爆单自动调度其他区域
  - 全程自动化，不用你盯着

# 6. 一句话终极总结

**K8s 就是管理 Docker 容器的自动化运维平台，
让大规模应用自动部署、自动重启、自动扩缩容、永不宕机。**
